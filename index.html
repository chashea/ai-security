Microsoft Entra

Identity & Access Security
Capabilities Signal Flow

Conditional Access for AI Policy

Per-request policy evaluation targeting AI app registration IDs. Conditions: user risk (Identity Protection ML), device compliance (Intune), named location, client app type. Grants: phishing-resistant authentication strength, compliant device, Terms of Use. Session controls: sign-in frequency, CAE, Token protection.

Signal: SigninLogs // Grant: authenticationStrength + compliantDevice
Target: Azure OpenAI App IDsignInRiskLevels: [medium, high]Session: CAE + Token protection

Workload Identity Federation Non-Human

OIDC token exchange eliminates client secrets for AI agents. Federate from any OIDC-compliant issuer (GitHub Actions, Kubernetes, GCP, AWS) for Entra token issuance. Conditional Access for workload identities (Workload ID Premium) gates service principal access by IP range and federation issuer.

federatedIdentityCredential resource // POST /applications/{id}/federatedIdentityCredentials
Issuers: GitHub, K8s, GCP, AWS, OIDCCA for workload identitiesCredential expiration alerts

Permissions Management (CIEM) Multi-Cloud

Permission Creep Index (PCI) scoring for every AI service principal across Azure, AWS, GCP. Quantifies granted vs. used permissions gap. Auto-remediation right-sizes RBAC roles to actual observed API call patterns over a 90-day window. JIT elevation for break-glass via PIM.

PCI: 0-100 per identity // Auto-right-size after 90-day observation
Azure + AWS + GCPAuto-right-size: custom roleJIT: PIM on-demand elevation

Identity Protection Real-Time

ML-based risk detections for AI-accessing identities: atypical travel, unfamiliar sign-in properties, and anomalous token characteristics (lifetime, binding, replay markers) — applies to all token use including calls into AI/agent endpoints. Feeds risk scores to Conditional Access for automatic enforcement.

Signal: riskyUsers, riskyServicePrincipals // Risk: low/medium/high
mcasImpossibleTravelunfamiliarFeaturesanomalousToken

Continuous Access Evaluation Zero Trust

Near-real-time token revocation on critical events: user disabled/deleted, password change, admin token revocation, high user risk from Identity Protection, and IP location change (with strict location enforcement). No waiting for token expiry. Claims challenge forces re-authentication within minutes instead of hours.

Claims challenge: xms_cc=cp1 // Token lifetime: up to 28h with CAE
User disabled / password changeAdmin token revocationHigh user risk + IP change

Identity Governance Lifecycle

Automated access reviews for AI resource RBAC assignments. Entitlement management packages for self-service AI access with multi-level approval workflows. Lifecycle workflows auto-revoke AI access on role change or offboarding.

API: identityGovernance/accessReviews // Lifecycle Workflows
Access reviews: quarterlyEntitlement packagesAuto-revoke on offboard

Defender for Cloud

Cloud Security Posture & AI Threat Protection
Capabilities Signal Flow

AI Security Posture (CSPM) Posture

Cloud Security Graph continuously discovers AI workloads across Azure OpenAI Service, Azure AI Foundry, and Azure Machine Learning — plus Amazon Bedrock and Google Vertex AI on the multi-cloud side (with Azure AI Foundry Agents and Copilot Studio Agents in preview). MCSB-aligned recommendations (Private Link, restrict network access, disable local auth, diagnostic logs) feed Secure Score; preview Foundry/agent-specific recommendations are surfaced but don’t yet count toward Secure Score.

Plan: Defender CSPM // Multicloud: Azure + AWS + GCP
AOAI, Foundry, AMLBedrock + Vertex AIMCSB-aligned → Secure Score

Attack Path Analysis Critical

Graph-based chained vulnerability analysis leading to AI resources. Example: Internet-exposed VM → managed identity with Contributor → Key Vault containing AOAI keys → model endpoint. Prioritized by blast radius and exploitability with per-node remediation guidance.

Filters: risk level, asset type, remediation status, time frame // Group by: attack path name, entry point, target type, target criticality
Internet-exposed AOAI → over-privileged MI → Key VaultEntry point → identity → target graphBlast radius visualization (Exposure Mgmt)

AI Runtime Threat Protection Real-Time

Behavioral detections on Azure OpenAI and Azure AI services API calls: jailbreak prompt injection (via Prompt Shields), ASCII smuggling, suspected wallet attacks (volume anomaly + duplicate-request patterns vs historical usage), Tor / anonymized IP access, Microsoft Threat Intelligence–flagged suspicious IPs and user agents, credential theft in model responses, and anomalous tool invocations. Alerts feed directly into Defender XDR incidents.

Surfaces in: SecurityAlert table (Log Analytics / Sentinel) + Defender XDR portal // MITRE ATT&CK tactics mapped per alert

Capabilities Signal Flow

Unified Incident Graph Correlation

AI threat alerts from Defender for Cloud automatically join the XDR incident graph alongside identity (Entra), endpoint (MDE), email (MDO), and cloud app (MDCA) signals. Single incident with full cross-workload attack timeline and MITRE ATT&CK mapping.

API: Microsoft Graph /security/incidents // Cross-workload correlation engine
IncidentId → AlertEvidence joinCross-workload: MDC+MDE+MDO+MDCAAuto-enrichment

Advanced Hunting KQL

KQL queries across all security tables for AI-specific pattern detection. Build custom detection rules for token anomalies, cross-department grounding access, and suspicious process behavior on AI endpoints. 30-day data retention for threat investigation.

Tables: CloudAppEvents, EntraIdSpnSignInEvents, DeviceEvents, AlertEvidence
CloudAppEventsDeviceEvents + AlertEvidenceCustom detection rules

MDCA — AI SaaS Discovery CASB

Cloud App Discovery identifies hundreds of Generative AI SaaS apps from network logs with risk scoring per app. Conditional Access app control enables real-time session policies: block file uploads to unsanctioned AI apps, apply Purview sensitivity labels on download, enforce DLP scanning on AI-bound traffic, step-up MFA for sensitive operations.

Discovery: CloudAppEvents // Session policies: Conditional Access app control
Generative AI app catalogApp Governance: OAuth consentRisk scoring per AI app

Automated Investigation & Response AIR

Playbook-driven response for AI incidents: disable compromised service principal, isolate or contain endpoint, force password reset and revoke refresh tokens. Investigation graph traces full attack chain with recommended containment actions.

Microsoft Purview

Data Security & AI Governance
Capabilities Signal Flow

Apps & Agents (DSPM for AI) Observability

The Apps & Agents view inside DSPM for AI helps inventory supported AI apps and agents — including Microsoft 365 Copilot, Copilot agents, and third-party generative AI sites discovered through Defender for Cloud Apps. It highlights risky AI usage, sensitive data exposure, and remediation recommendations from supported Purview and Defender signals.

Portal: purview.microsoft.com → Solutions → DSPM for AI // Source: MDCA discovery + supported Purview signals
Apps & Agents inventoryRisk indicatorsRemediation recommendations

DLP for AI Interactions Enforcement

Purview DLP can apply controls to Microsoft 365 Copilot and supported endpoints to reduce risky AI interactions, such as prompts containing sensitive information or use of labeled content where policy excludes it from Copilot processing. Browser DLP on managed endpoints and MDCA session controls can extend protection to supported third-party generative AI sites.

Locations: Microsoft 365 Copilot, Devices, Defender for Cloud Apps // Actions depend on policy and workload support
Sensitive prompt controlsLabel-aware exclusionsEndpoint + MDCA coverage

Sensitivity Labels & Auto-Labeling Classification

Microsoft 365 Copilot respects existing Microsoft 365 permissions and Purview sensitivity labels on source content. Labeled SharePoint and OneDrive files keep their classification and protection at rest, and supported Office experiences can apply labels to generated content based on policy. Auto-labeling policies can classify content using sensitive information types and trainable classifiers.

Copilot honors source permissions and labels // Auto-labeling: SIT + trainable classifier support
Label-aware Copilot controlsAuto-label: SIT + classifierPermission-trimmed grounding

Data Risk Assessments (DSPM) Posture

Inside DSPM for AI, Data Risk Assessments scan SharePoint and OneDrive sites for overshared content Copilot could surface — files with "Everyone except external users" or broad permissions containing sensitive info types or labels. Guided remediation flows recommend restricting sharing, applying labels, or running site access reviews before Copilot rollout.

Surface: DSPM for AI → Data risk assessments // Scope: SharePoint + OneDrive sites
Oversharing assessmentGuided remediationPre-Copilot data hygiene

Insider Risk Management Behavioral

The "Risky AI usage" policy template surfaces indicators for risky prompts containing sensitive information, sensitive responses in AI apps, and prompts containing harmful content. If HR connectors and Adaptive Protection are configured, those signals can help tune risk levels and DLP policy scope.

Template: Risky AI usage // Optional enrichment: HR connectors + Adaptive Protection
Risky prompt indicatorsSensitive response indicatorsOptional HR enrichment

Audit & eDiscovery Compliance

Unified audit logs the occurrence of every Copilot interaction with rich metadata — user, time, app host, AccessedResources (with SensitivityLabelId and XPIADetected fields) — but not the prompt or response text itself. Actual prompt and response content is retained as a compliance copy in a hidden mailbox folder and retrieved through eDiscovery (Content Search or Premium) for legal hold and investigations.

Operation: CopilotInteraction // Retention: Audit Standard 180d, Premium 1yr // eDiscovery: Exchange mailbox, Copilot interactions type
Audit: metadata onlyeDiscovery: full contentHidden mailbox folder

Governance for Supported AI Agents
Capabilities Signal Flow

Agent Registry Inventory

Central inventory for supported AI agents across the organization, including agents built in Microsoft tools and connected partner ecosystems where supported. Agent records can include owner, purpose, lifecycle state, data sources, tools, and operating environment so administrators can govern usage consistently.

Surface: Microsoft 365 admin experiences // Identity: Entra Agent ID where supported
Supported agent inventoryOwner attributionLifecycle metadata

Access Control Identity

Use Entra Agent ID, workload identities, federated credentials, and least-privilege permissions to scope what supported agents can access. Conditional Access for workload identities and Identity Governance controls can help reduce credential exposure and permission creep where those controls apply.

Policy: Entra workload identities // OIDC federated credentials // Least privilege
Workload identity controlsJust-enough-permissionsFederated credentials

Agent Visualization Graph

Agent relationship views can help administrators understand supported agent connections to people, data, tools, and other agents. Use topology and ownership context to review blast radius, find stale agents, and prioritize access reviews.

Visualization: supported admin center graph views // Review: ownership + access paths
Topology contextData/tool bindingsAccess review candidates

Interoperability Open Standards

Microsoft is moving toward standards-based agent interoperability, including MCP and agent-to-agent patterns. Treat non-Microsoft framework governance as connector- and capability-dependent: register and govern agents through supported integration paths rather than assuming universal coverage.

Protocols: MCP + agent-to-agent patterns // Coverage depends on supported connectors
Standards alignmentConnector-based integrationFramework-aware governance

Agent Security Threat Detection

Agent activity can contribute to security investigations through supported Defender, Purview, and Entra signals. Detections and coverage vary by agent platform and connector, so validate which activities are logged before relying on a scenario for detection engineering.

Sources: supported Defender XDR, Purview, Entra, and agent-platform logs

Admin Center Surface Unified

Use Microsoft 365 admin and Entra governance surfaces to manage supported agent lifecycle tasks: review ownership, scope access, monitor usage, and deprovision stale agents. Pair this with periodic access reviews for high-privilege agent identities and data sources.

Threat Scenarios Implementation Roadmap
HIGH Shadow AI Data Leak
Employee copies customer PII (Social Security numbers, financial records) and pastes it into an unsanctioned AI chatbot for analysis
STEP 1
Shadow AI Discovered
MDCA
Cloud App Discovery identifies the unsanctioned AI app from network logs. App risk score: 3/10 — no SOC 2, weak encryption, broad data retention.
STEP 2
Sensitive Data Blocked
Purview DLP
Endpoint DLP detects SSN and financial SIT patterns in clipboard data bound for browser AI domain. Policy action: block paste + user notification.
STEP 3
Behavioral Anomaly
Insider Risk
Insider Risk Management detects repeated sensitive data copy events + unusual AI usage. If HR connectors are configured, those signals can enrich the risk context.
STEP 4
Incident Correlated
Defender XDR
DLP alert + Insider Risk signal + CloudAppEvents join the incident graph. Single incident with full timeline and recommended response.
CRITICAL Prompt Injection on Microsoft Foundry Agents
External attacker hides indirect prompt-injection instructions in a document a Microsoft Foundry Agent ingests through a grounding source or connector, attempting to coerce the agent into invoking tools, exfiltrating system instructions, or leaking grounded data
STEP 1
Injection Detected
Content Safety
Azure AI Content Safety Prompt Shield identifies indirect injection pattern. Jailbreak filter catches embedded instruction to ignore system prompt.
STEP 2
Threat Alert Fired
Defender for Cloud
Defender for Cloud AI threat protection can raise a supported AI-service alert, such as a Prompt Shields jailbreak alert for Foundry agents, with context that helps analysts investigate the attempted injection.
STEP 3
Attack Chain Mapped
Defender XDR
Alert joins XDR incident graph. Advanced Hunting correlates source IP across sign-in logs and CloudAppEvents — reconnaissance pattern identified.
STEP 4
Session Terminated
Entra CAE
Continuous Access Evaluation receives critical event signal. Active tokens revoked within minutes. CA policy enforces re-auth with phishing-resistant MFA.
MEDIUM Overshared Data via Copilot
Sensitive HR compensation data with “Everyone” permissions on SharePoint is surfaced in Microsoft Copilot responses to unauthorized employees
STEP 1
Oversharing Detected
Purview DSPM
DSPM scans SharePoint and flags files with “Everyone” permissions containing sensitive info types (salary data, SSNs, performance ratings).
STEP 2
Labels Applied
Sensitivity Labels
Auto-labeling detects PII patterns and applies a “Highly Confidential — HR Only” label according to policy. Copilot honors source permissions and label-based controls during supported grounding and response experiences.
STEP 3
Copilot Blocked
Purview DLP
DLP policy for Copilot location blocks surfacing Highly Confidential content to users outside HR security group. Policy tip shown to requesting user.
STEP 4
Bulk Query Flagged
Insider Risk
Insider Risk detects repeated Copilot queries targeting compensation data across departments. Risk score elevated; adaptive protection triggers stricter DLP scope.
CRITICAL Rogue Agent — Excessive Tool Invocation
A Copilot Studio agent built by a citizen developer accumulates broad data-source bindings over time and begins invoking high-privilege tools far outside its declared purpose, exfiltrating SharePoint content to an external connector
STEP 1
Agent Inventoried
Agent 365
Agent Registry shows the agent with its Agent ID, owner, declared purpose, and current bindings: 14 data sources + 6 tools (well above peer agent baseline of 3 + 2).
STEP 2
Topology Anomaly
Agent Visualization
Graph view flags the agent as a hub with high blast radius. Recent edge: new outbound connector to an external SaaS endpoint added 3 days ago by the owner.
STEP 3
Data Exfil Detected
Purview DSPM
DSPM for AI flags bulk reads of "Confidential" labeled SharePoint files by the agent, followed by transmission to the new connector. DLP policy blocks the egress.
STEP 4
Agent Quarantined
Defender XDR
Response teams use the incident context to disable or scope the agent identity, revoke exposed credentials where applicable, and review topology and data-flow evidence from supported admin and security portals.
CRITICAL Compromised AI Agent Identity
Attacker obtains an overprivileged service principal credential used by an AI agent, then pivots through Key Vault to access the Azure OpenAI model endpoint
STEP 1
Anomalous Sign-In
Identity Protection
Entra Identity Protection detects service principal sign-in from unfamiliar IP + impossible travel. Risk level: high. Flagged as RiskyServicePrincipal.
STEP 2
Overprivilege Found
Entra CIEM
Permissions Management shows PCI 92/100 — SP has Contributor on subscription but only uses 3 AOAI API operations. Massive permission gap.
STEP 3
Attack Path Mapped
Defender for Cloud
Attack path: Compromised SP → Contributor RBAC → Key Vault (AOAI keys) → Azure OpenAI endpoint. Blast radius: all model deployments + training data.
STEP 4
Auto-Remediation
Defender XDR
Auto-Investigation disables compromised SP, rotates Key Vault secrets, blocks attacker IP. Full incident timeline with 4-product evidence chain.
CRITICAL Compromised Endpoint Device Accessing Copilot
An employee laptop is compromised via a malicious browser extension that exfiltrates session cookies and silently relays Microsoft 365 Copilot prompts and responses — including grounded sensitive data — to an attacker-controlled endpoint
STEP 1
Device Compromise Detected
Defender for Endpoint
MDE EDR flags the malicious browser extension behavior — cookie theft, unsigned process injection, beaconing to a low-reputation domain. Device risk level escalates to High and an alert fires with full process tree.
STEP 2
Compliance Flipped & Access Blocked
Intune + Entra CA
Intune consumes the MDE risk signal and marks the device non-compliant. Entra Conditional Access for AI apps requires compliant device — Copilot and Foundry access are blocked, and CAE revokes existing tokens within minutes.
STEP 3
Data Exposure Assessed
Purview DSPM for AI
Purview and audit signals help assess recent Copilot activity, sensitive data indicators, labels involved, and related DLP or Insider Risk events scoped to the affected user and device.
STEP 4
Incident Correlated & Contained
Defender XDR
XDR joins the MDE alert, Conditional Access events, available Copilot audit metadata, and Purview risk signals into one incident. Analysts can use automated investigation actions to isolate the device and remove the malicious extension.
